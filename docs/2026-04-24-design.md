# Neovim 0.12 Config Replica — Design Spec

- **Date**: 2026-04-24
- **Owner**: jc (fangdasenclassic@gmail.com)
- **Source config**: `~/.config/nvim` (fork of ayamir/nvimdots, branch `0.11`)
- **Target directory**: `~/.config/nvim-0.12/`
- **Launch command**: `NVIM_APPNAME=nvim-0.12 nvim`

## 1. Goal

Replicate the user's current Neovim workflow into a self-contained config that:

1. Targets Neovim **0.12** (nightly), using native APIs for LSP / completion / diagnostics.
2. **Fully detaches** from the upstream `ayamir/nvimdots` layered architecture
   (no `modules/` ↔ `user/` split).
3. **Reduces plugin footprint** — the number of top-level specs I author drops
   from ~65 to ~37 (≥40% drop); total resolved plugins in `lazy-lock.json`
   drops from ~120 to ~70 (≥40% drop). Keeps only plugins the user actually uses.
4. Keeps **lazy.nvim** as the plugin manager (hybrid path — not `vim.pack`).
5. Replaces `nvim-cmp` with **blink.cmp**, and `none-ls` with **conform.nvim +
   nvim-lint**.
6. Managed LSP/formatter/linter binaries via **system / mise**, not `mason.nvim`.

## 2. Non-Goals

- Not migrating to `vim.pack` (lazy.nvim retained).
- Not adopting LazyVim/NvChad/kickstart distributions.
- Not merging or preserving the upstream `modules/` layer — everything is authored
  fresh in the new directory.
- Not NixOS (user confirmed).

## 4. Bootstrap Flow

```
init.lua
  └── require("core")
        └── core/init.lua
              1. require("core.globals")
              2. require("core.options")
              3. require("core.keymaps")
              4. require("core.autocmds")
              5. require("core.lazy")         -- installs/loads lazy.nvim
                     └── { import = "plugins" }  -- loads plugins/*.lua
              6. require("configs.lsp")        -- vim.lsp.config + vim.lsp.enable
```

LSP setup runs last so that `vim.lsp.enable` is called after all plugins (including
`nvim-lspconfig` default server definitions) are registered.

## 5. Native API Migration Detail

### 5.1 LSP

```lua
-- lua/configs/lsp/init.lua
local servers = require("core.settings").lsp_servers
for _, name in ipairs(servers) do
  local ok, overrides = pcall(require, "configs.lsp." .. name)
  vim.lsp.config(name, ok and overrides or {})
end
vim.lsp.enable(servers)

-- Shared on_attach via LspAttach autocmd (single source of truth)
vim.api.nvim_create_autocmd("LspAttach", {
  callback = function(args)
    local bufnr = args.buf
    -- keymaps: gd/gr/gi/K/<leader>ca/<leader>rn ...
    -- trigger native completion if capability present
    local client = vim.lsp.get_client_by_id(args.data.client_id)
    if client and client.server_capabilities.completionProvider then
      -- blink.cmp handles this; we rely on it, no native fallback
    end
  end,
})
```

Per-server override files (e.g., `configs/lsp/basedpyright.lua`) return a plain
table merged into the default config.

### 5.2 Completion — blink.cmp

Single plugin; replaces nvim-cmp + 8 sources + cmp-under-comparator.

```lua
-- lua/configs/completion/blink-cmp.lua
return {
  keymap = { preset = "default" },            -- <C-space>/<Tab>/<CR>
  appearance = { nerd_font_variant = "mono" },
  completion = {
    accept = { auto_brackets = { enabled = true } },
    menu = { draw = { treesitter = { "lsp" } } },
    documentation = { auto_show = true, auto_show_delay_ms = 200 },
  },
  sources = { default = { "lsp", "path", "snippets", "buffer" } },
  snippets = { preset = "luasnip" },
  fuzzy = { implementation = "prefer_rust_with_warning" },
}
```

### 5.3 Diagnostics

```lua
-- in core/options.lua or a diagnostics autocmd
vim.diagnostic.config({
  virtual_text = false,
  virtual_lines = { current_line = true },    -- 0.11+ native inline
  signs = true,
  underline = true,
  update_in_insert = false,
  severity_sort = true,
  float = { border = "rounded", source = "if_many" },
})
```

### 5.4 Formatting — conform.nvim

```lua
-- lua/configs/formatters/init.lua (plugin opts)
return {
  formatters_by_ft = {
    lua = { "stylua" },
    python = { "ruff_format", "ruff_organize_imports" },
    c = { "clang-format" },
    cpp = { "clang-format" },
    sh = { "shfmt" },
    javascript = { "prettier" },
    typescript = { "prettier" },
    json = { "prettier" },
    yaml = { "prettier" },
    markdown = { "prettier" },
    tex = { "latexindent" },
  },
  format_on_save = function(bufnr)
    if vim.g.disable_autoformat or vim.b[bufnr].disable_autoformat then return end
    return { timeout_ms = 2000, lsp_format = "fallback" }
  end,
}
```

`<leader>tf` toggles `vim.g.disable_autoformat`.

### 5.5 Linting — nvim-lint

```lua
-- lua/configs/linters/init.lua
local lint = require("lint")
lint.linters_by_ft = {
  vim = { "vint" },
  sh = { "shellcheck" },
  -- python handled by ruff LSP; no duplicate lint needed
}
vim.api.nvim_create_autocmd({ "BufWritePost", "BufReadPost", "InsertLeave" }, {
  callback = function() lint.try_lint() end,
})
```

## 8. Settings Surface (`core/settings.lua`)

Preserve the user's existing flags:

```lua
return {
  colorscheme = "catppuccin-latte",
  background = "dark",
  use_ssh = false,
  use_copilot = false,
  use_chat = false,             -- codecompanion toggle

  lsp_servers = {
    "lua_ls", "basedpyright", "ruff", "clangd", "bashls",
    "html", "jsonls", "yamlls", "texlab", "ltex",
  },

  server_formatting_block_list = {
    lua_ls = true, clangd = true, texlab = true,
    basedpyright = true, pyright = true,
  },

  treesitter_deps = {
    "bash", "c", "cpp", "css", "go", "gomod", "html", "javascript",
    "json", "lua", "make", "markdown", "markdown_inline", "python",
    "rust", "typescript", "vimdoc", "vue", "yaml", "ini", "r",
    "rnoweb",
  },

  gui_config = { font_name = "Iosevka NFP", font_size = 11 },
  neovide_config = { fullscreen = false },

  chat_models = {
    "anthropic/claude-3.7-sonnet",
    "google/gemini-2.5-pro-preview",
  },

```

Removed from old `settings.lua`:
- `null_ls_deps` — conform/nvim-lint have their own declarative config
- `lsp_deps` — superseded by `lsp_servers`
- `disabled_plugins` — new config has no upstream layer, so nothing to disable

## 10. Migration Execution Order

1. Scaffold directory tree + `init.lua` + `core/*`.
2. Bootstrap lazy.nvim in `core/lazy.lua`.
3. Author `plugins/ui.lua` + `configs/ui/*` → get colorscheme + lualine loading.
4. Author `plugins/editor.lua` + treesitter config → get syntax highlighting.
5. Author `plugins/completion.lua` + `configs/lsp/*` + blink.cmp config → LSP + completion live.
6. Author `plugins/tool.lua` → telescope, tree, terminal, dap, harpoon.
7. Author `plugins/lang.lua` → vimtex, rustaceanvim, markdown.
8. Copy `luasnippets/`, `snips/`, `templates/`, `stylua.toml`.
9. Smoke-test with `NVIM_APPNAME=nvim-0.12 nvim` on representative files
   (Python, Rust, LaTeX, Lua, Markdown).
10. Iterate on keymap + config ergonomics.

## 11. Risks / Open Questions

- **0.12 is nightly**: some plugins may not yet be compatible. Expected casualties
  to watch: `nvim-treesitter` main branch config API drift, `blink.cmp` upstream
  churn. Mitigation: pin via `lazy-lock.json` after first green.
- **fcitx5.nvim** is lightly maintained. If it breaks, fallback is a 10-line
  InsertLeave/InsertEnter autocmd wrapping `fcitx5-remote`.
- **Keymap parity**: the old config spreads keymaps across `lua/user/keymap/`. The
  exact translation into `core/keymaps.lua` (potentially split by topic file) will
  be settled during implementation, not here — tracked as a planning task.
- **codecompanion config** is complex (history, utilities). Retained gated by
  `use_chat`; verified working in old config, expected to port cleanly.

## 12. Success Criteria

- `NVIM_APPNAME=nvim-0.12 nvim` starts in ≤80ms on this machine.
- Opening a `.py`, `.rs`, `.tex`, `.lua`, `.md` file yields: syntax highlight, LSP
  attach, diagnostics, completion popup, format-on-save.
- No `nvim-cmp`, no `mason`, no `none-ls`, no `lspsaga` in `lazy-lock.json`.
- Total plugin count in `lazy-lock.json` ≤ 75 (including transitive deps).

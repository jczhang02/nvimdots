# Neovim 0.12 Config Replica — Implementation Plan

Spec: `docs/2026-04-24-design.md`
Target root: `~/.config/nvim-0.12/`
Source for porting references: `~/.config/nvim/` (old config)

## Ground Rules

- All file paths are **absolute** under `~/.config/nvim-0.12/`.
- **Never** touch `~/.config/nvim/` except to read reference files.
- **Port, don't copy verbatim** — drop upstream nvimdots scaffolding (`lua/core/pack.lua`,
  `lua/core/event.lua` style, `lua/modules/`, `lua/user/` layering).
- Use **lazy.nvim** as plugin manager; bootstrap in `lua/core/lazy.lua`.
- Use **blink.cmp** not nvim-cmp; **conform.nvim** + **nvim-lint** not none-ls.
- Use native `vim.lsp.config` + `vim.lsp.enable` for LSP; no Mason.
- When a plugin's old config file exists in `~/.config/nvim/lua/modules/configs/...`
  or `~/.config/nvim/lua/user/configs/...`, read it as reference and simplify.
- Every plugin spec uses `lazy = true` with concrete `event` / `cmd` / `ft` / `keys`
  lazy-load triggers **unless the plugin requires eager load** (e.g. colorscheme,
  treesitter main branch, dropbar).
- Keymaps: for now dump them into `lua/core/keymaps.lua`; topic-split only if the
  file grows past ~200 lines.
- No README.md, no LICENSE — the user owns this config privately.

## Task Graph

Tasks are grouped into stages. Within a stage, tasks are **independent** and can
run in parallel. Stages are **sequential** (later stages depend on earlier ones).

### Stage 0 — Scaffold (sequential, 1 task)

**T0**: Create the directory tree and empty `init.lua`.

- Create dirs:
  - `lua/core/`
  - `lua/plugins/`
  - `lua/configs/completion/`
  - `lua/configs/editor/`
  - `lua/configs/lang/`
  - `lua/configs/tool/`
  - `lua/configs/ui/`
  - `lua/configs/lsp/`
  - `lua/configs/formatters/`
  - `lua/configs/linters/`
  - `lua/utils/`
- Write `init.lua`:
  ```lua
  if not vim.g.vscode then
    require("core")
  end
  ```
- Copy `stylua.toml` from old config.

### Stage 1 — Core bootstrap (parallelizable, 6 tasks)

**T1a — `lua/core/globals.lua`**: OS detection, paths (`vim_path`, `data_dir`,
`cache_dir`), disable built-in providers (perl, ruby, node) for speed.

**T1b — `lua/core/options.lua`**: Port from `~/.config/nvim/lua/core/options.lua`.
Drop anything nvimdots-specific. Include 0.12-friendly defaults (`winborder`,
`smoothscroll`). Set leader keys at the top (before lazy).

**T1c — `lua/core/settings.lua`**: The user-tunable settings table (see spec §8).

**T1d — `lua/core/autocmds.lua`**: Port from `~/.config/nvim/lua/core/event.lua`.
Keep: highlight-on-yank, auto-close qf, restore cursor, LSP-attach skeleton
(actual LspAttach callback lives in `configs/lsp/init.lua`).

**T1e — `lua/core/keymaps.lua`**: Port from `~/.config/nvim/lua/keymap/*` and
`~/.config/nvim/lua/user/keymap/*`. Merge non-plugin-specific keymaps only.
Plugin-specific keymaps go in each plugin's `keys = {}` spec.

**T1f — `lua/core/lazy.lua`**: lazy.nvim bootstrap (clone if missing), then
`require("lazy").setup({ { import = "plugins" } }, { ...opts... })`. Configure
performance-oriented lazy opts: disable `rtp.reset = true` plugins that aren't
used, set `change_detection = { enabled = false }`.

**T1g — `lua/core/init.lua`**: Orchestrator — requires in order: globals,
options, keymaps, autocmds, lazy, then defers `configs.lsp` via VeryLazy event.

### Stage 2 — UI plugins (parallelizable, 9 tasks)

All live in `lua/plugins/ui.lua` (single return table) + per-plugin config files.

**T2a — catppuccin** (`lua/configs/ui/catppuccin.lua`): latte flavor, integrations
for treesitter/telescope/which-key/gitsigns/indent-blankline/bufferline/dap/dap-ui/
neogit/trouble/which-key/lsp-trouble. Call `colorscheme catppuccin-latte` as part
of config after setup.

**T2b — lualine** (`lua/configs/ui/lualine.lua`): catppuccin theme, sections
for mode/branch/diff/diagnostics/filename/filetype/progress/location. Keep
lightweight (no custom components).

**T2c — bufferline** (`lua/configs/ui/bufferline.lua`): catppuccin integration,
always-show, separator style thin, diagnostics `nvim_lsp`.

**T2d — alpha** (`lua/configs/ui/alpha.lua`): dashboard theme, header ASCII art
(copy from old), buttons: find file, recent, config, lazy, quit.

**T2e — gitsigns** (`lua/configs/ui/gitsigns.lua`): default signs, current_line_blame
on, keymaps via on_attach.

**T2f — indent-blankline** (`lua/configs/ui/indent-blankline.lua`): scope enabled,
exclude alpha/help/terminal filetypes.

**T2g — nvim-notify** (`lua/configs/ui/notify.lua`): `vim.notify =
require("notify")`, timeout 2500ms.

**T2h — smart-splits** (`lua/configs/ui/smart-splits.lua`): setup default;
keymaps `<C-h/j/k/l>` for navigate, `<A-h/j/k/l>` for resize.

**T2i — todo-comments** (`lua/configs/ui/todo-comments.lua`): default keywords,
telescope integration via `:TodoTelescope`.

### Stage 3 — Editor plugins (parallelizable, 6 + treesitter cluster)

Registered in `lua/plugins/editor.lua`.

**T3a — treesitter cluster** (`lua/configs/editor/treesitter.lua`): Use `main`
branch API. Install parsers from `settings.treesitter_deps`. Enable highlight,
indent, incremental_selection. Include deps in the same spec: textobjects,
matchup, ts-autotag, rainbow-delimiters, ts-context-commentstring. Config each
via its own tiny config file under `configs/editor/`.

**T3b — flash.nvim** (`lua/configs/editor/flash.lua`): `s`/`S` jump, `r` remote,
`R` treesitter_search, `<C-s>` toggle char mode in cmdline.

**T3c — nvim-surround** (`lua/configs/editor/nvim-surround.lua`): default.

**T3d — mini.align, autoclose, nvim-highlight-colors, suda, vim-sleuth,
persisted, diffview, grug-far, neogen, nvim-bufdel**: one file each when config
is non-trivial; inline plugin spec when the config is default.

### Stage 4 — Tool plugins (parallelizable, 8 tasks)

Registered in `lua/plugins/tool.lua`.

**T4a — telescope** (`lua/configs/tool/telescope.lua`): Port from old config
(`~/.config/nvim/lua/user/configs/telescope.lua` and modules version). Extensions:
fzf-native, live-grep-args, project, undo, zoxide. Keymaps on `<leader>f*`.

**T4b — which-key** (`lua/configs/tool/which-key.lua`): default v3 preset, add
`<leader>` group labels.

**T4c — nvim-tree** (`lua/configs/tool/nvim-tree.lua`): disable_netrw, filters,
git status icons, renderer icons from devicons.

**T4d — toggleterm** (`lua/configs/tool/toggleterm.lua`): float + horizontal,
`<leader>tt` toggle.

**T4e — trouble** (`lua/configs/tool/trouble.lua`): v3 API, open on big lists.

**T4f — dap cluster** (`lua/configs/tool/dap.lua`, `dapui.lua`): adapters for
python (debugpy) and codelldb. No mason-nvim-dap.

**T4g — codecompanion** (gated by `settings.use_chat`): port config from old
`user/configs/codecompanion.lua` and `codecompanion_utils/`.

**T4h — harpoon, smartyank, dropbar, asyncrun+vimux, asynctasks, gitignore,
debugprint, pangu, fcitx5.nvim**: inline or tiny config each.

### Stage 5 — Completion + LSP + Formatting + Linting (parallelizable, 4 tasks)

Registered in `lua/plugins/completion.lua`.

**T5a — blink.cmp** (`lua/configs/completion/blink-cmp.lua`): See spec §5.2.
LuaSnip preset, default keymap, lsp/path/snippets/buffer sources.

**T5b — LuaSnip** (`lua/configs/completion/luasnip.lua`): load `friendly-snippets`
via `require("luasnip.loaders.from_vscode").lazy_load()`. Load local snippets
from `~/.config/nvim-0.12/luasnippets/` via
`require("luasnip.loaders.from_lua").lazy_load({ paths = ... })`.

**T5c — LSP init** (`lua/configs/lsp/init.lua`): See spec §5.1. Per-server
overrides in `configs/lsp/<name>.lua`. Shared `on_attach` via LspAttach autocmd
sets keymaps (gd/gr/gi/K/`<leader>ca`/`<leader>rn`/`<leader>f`). Diagnostic
config via `vim.diagnostic.config` (spec §5.3). Capabilities merged with
`blink.cmp.get_lsp_capabilities()`.

**T5d — conform + nvim-lint** (`lua/configs/formatters/init.lua`,
`lua/configs/linters/init.lua`): See spec §5.4 and §5.5. `<leader>tf` toggles
`vim.g.disable_autoformat`.

### Stage 6 — Lang plugins (parallelizable, 8 tasks)

Registered in `lua/plugins/lang.lua`.

**T6a — vimtex**: port from `user/configs/lang/vimtex.lua`. Compile with latexmk,
pdf viewer zathura or similar (keep old config's choice).

**T6b — rustaceanvim**: port from `user/configs/rust.lua`. Pass `server.settings`
for rust-analyzer.

**T6c — crates**: default setup.

**T6d — render-markdown, markdown-preview, csv, nvim-bqf, python-type-stubs**:
tiny or default configs.

**T6e — luasnip-latex-snippets**: port from
`user/configs/completion/luasnip-latex-snippets.lua`. ft = `{ "tex", "bib",
"markdown" }`.

### Stage 7 — Assets + smoke test (sequential, 2 tasks)

**T7a — Copy assets** (single task):
- `cp -r ~/.config/nvim/luasnippets ~/.config/nvim-0.12/`
- `cp -r ~/.config/nvim/snips ~/.config/nvim-0.12/`
- `cp -r ~/.config/nvim/templates ~/.config/nvim-0.12/`

**T7b — Smoke test**: Run `NVIM_APPNAME=nvim-0.12 nvim --headless -c "quitall"` to

`docs/2026-04-24-smoke-errors.md` but do NOT attempt interactive fixes (user will

### Stage 8 — Finalize (sequential, 1 task)

**T8 — Git init**:
- `cd ~/.config/nvim-0.12 && git init`
- Write `.gitignore` (ignore `lazy-lock.json.old`, swp files)
- Initial commit: "feat: initial nvim 0.12 config (replica of old config)".

## Parallelization Strategy for ultrawork

Stage 0 must finish before Stage 1. Stages 1–6 tasks within a stage are
parallel. Stage 7 runs after Stage 6. Stage 8 runs last.

Recommended ulw grouping:
1. Run T0 alone.
2. Run all of Stage 1 in parallel (6 agents).
3. Run all of Stage 2 + Stage 3 + Stage 4 + Stage 5 + Stage 6 in parallel
   (~30 agents grouped by file). If agent budget is tight, batch by stage.
4. Run Stage 7 + Stage 8 sequentially.
